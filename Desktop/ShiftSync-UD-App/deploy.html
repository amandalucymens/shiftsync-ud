Deploy: put ShiftSync UD on the internet

Right now, your app runs on your laptop. Deployment means putting it on a server that’s always on, so anyone with the link can use it—even when your computer is closed.

Mental model: from “local” to “live”

  1. GitHub stores your code (like a shared folder that Vercel can access).
  2. Vercel reads your GitHub repo, builds the app, and hosts it at a URL.
  3. When you push changes to GitHub, Vercel automatically redeploys (usually in 1–2 minutes).

1) Push your code to GitHub

Why: Vercel needs a GitHub repo to pull your code from. Also, GitHub gives you a backup and version history.

  1. Go to github.com and sign in.
  2. Click the + (top right) → New repository.
  3. Name it something like shiftsync-ud (lowercase, dashes).
  4. Keep it Public (simplest for free hosting).
  5. Don’t initialize with README or .gitignore (your project already has files).
  6. Create the repository.

Then, in Cursor…

Open the Source Control panel, or use the terminal inside Cursor. If you’re new to git, the simplest move is to ask Cursor directly: 

Please help me push this project to GitHub. My repository URL is: [PASTE YOUR REPO URL]

2) Deploy on Vercel

Why Vercel: it’s built for Next.js and handles builds, HTTPS, and redeploys automatically.

  1. Go to vercel.com and sign in.
  2. Click Add New → Project.
  3. Select your GitHub repo for ShiftSync UD.
  4. When prompted, add an environment variable:
Name Value Why it matters
NEXT_PUBLIC_INSTANT_APP_ID your InstantDB App ID Tells your app which InstantDB project to use.
  1. Click Deploy and wait for the build to finish.
  2. Open the live URL Vercel gives you.

What are environment variables?

Environment variables are values you configure outside your code. They’re a clean way to set things like “which database should this app talk to?” without hardcoding it in public files.

The NEXT_PUBLIC_ prefix means it’s safe to expose to the browser (and in this case, InstantDB App IDs are public anyway).

3) Push your InstantDB schema + permissions

Your app has two “rulesets” that live in code: a schema (what data exists) and permissions (who can read/write). InstantDB needs you to “push” these so the hosted backend enforces them.

Cursor may already add the right commands. If you’re unsure, ask Cursor: 

How do I push my InstantDB schema and permissions for this project? Please give me the exact commands to run.
