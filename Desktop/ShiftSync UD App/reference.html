Reference: next steps + troubleshooting

This page is your “keep going” guide. It shows how to make changes after your first version is live, what to build next for ShiftSync UD, and how to debug common issues without feeling stuck.

How to make changes to your app (the loop)

Mental model: you’re shipping small improvements, like updating a menu. You don’t redesign the whole restaurant every time—you change one thing, test it, then publish it.

  1. Make the change in Cursor (ask the AI if you want).
  2. Test locally: run npm run dev.
  3. Commit + push to GitHub.
  4. Wait 1–2 minutes: Vercel redeploys automatically.

Troubleshooting (FAQ)

Click a question to expand it. If you still feel stuck, copy the exact error text and paste it into Cursor.

I see a blank page

›
  • Open your browser console: right-click → Inspect → Console tab.
  • Copy the red error message (or the stack trace).
  • Paste it into Cursor and ask: “I see a blank page. Fix this error.”

`npm run dev` shows an error

›
  • First try: run npm install (it installs missing packages).
  • If it still fails, copy the full error output and paste it into Cursor.
  • Ask: “I’m getting this error when running npm run dev. Please fix it.”

My app works locally but not on Vercel

›
  • Check Vercel build logs (Project → Deployments → click latest).
  • Confirm env var is set: NEXT_PUBLIC_INSTANT_APP_ID.
  • Make sure you pushed all changes to GitHub (the repo is the source of truth).

Login isn’t working (Magic Code)

›
  • Double-check your InstantDB App ID is correct.
  • Check spam/junk folder for the code email.
  • Try a different email address to rule out deliverability issues.
  • Confirm you pushed schema/perms if your app is failing with permissions errors.

I can’t access the manager page

›

In V1, manager access is usually controlled by a role field like role="manager" on your profile.

  • Open your Profile page and confirm your role is set correctly.
  • If your UI doesn’t allow role changes, ask Cursor to add a safe way to set it (for a demo).
  • If permissions block it, ask Cursor to update instant.perms.ts.

My changes aren’t showing up on the live site

›
  • Wait 1–2 minutes (Vercel redeploy takes time).
  • Hard refresh: Ctrl+Shift+R.
  • Confirm you pushed the commit to GitHub.
