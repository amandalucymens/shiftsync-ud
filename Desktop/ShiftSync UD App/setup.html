Setup: accounts + tools

This page is the “lay the foundation” step. The goal is to get your accounts created and your computer ready so that building ShiftSync UD is smooth (and you don’t get stuck on confusing installation issues).

1) Create the accounts you’ll need

Mental model: building an app is like running a small restaurant. You need a place to store your “blueprints” (code), a place to “host the building” (deployment), and a “kitchen/pantry” (database + login).

GitHub (stores your code)

Why: GitHub holds your project files and version history. Vercel will pull your code from GitHub and deploy it automatically.

  1. Go to github.com and sign up.
  2. Pick a username you’d be comfortable sharing (it becomes part of your URLs).
  3. Verify your email.

Vercel (puts your app online)

Why: Vercel is the easiest way to deploy Next.js apps. The free “Hobby” plan is enough.

  1. Go to vercel.com.
  2. Sign up using GitHub and authorize access.

InstantDB (database + login)

Why: ShiftSync UD needs saved data and user accounts. InstantDB gives you both without building a custom backend server.

  1. Go to instantdb.com and sign up.
  2. Verify your email.
  3. Keep the dashboard link handy: instantdb.com/dash

2) Create your InstantDB app (and copy your App ID)

Mental model: your InstantDB App ID is like a mailing address for your database. Your Next.js app needs that address to know where to send “save” and “load” requests.

  1. Open the InstantDB dashboard: instantdb.com/dash
  2. Click Create App and name it something like shiftsync-ud.
  3. Copy the App ID (it looks like a long string with dashes).
  4. Paste it into a notes app for now—you’ll use it on the Build page.

Is the App ID secret?

No—InstantDB’s App ID is public. It’s not a password. Think of it like a public address. Security comes from InstantDB’s permissions rules, which control what logged-in users can read and write. 

Example App ID format:
566ffee5-e773-4b68-a132-29678f8f6eea

3) Install Node.js (so your computer can run app tools)

Mental model: during development, your computer needs to “run a mini version” of your app so you can preview it. Node.js is the tool that lets your computer run JavaScript outside the browser.

  1. Go to nodejs.org.
  2. Download the LTS version (stable, recommended).
  3. Run the installer and accept the defaults.

What is a “terminal” on Windows?

A terminal is a text-based way to give your computer instructions. On Windows, you’ll usually use PowerShell. You’ll only need a few simple commands, and Cursor will guide you.

Quick check (optional): confirm Node installed

Open PowerShell and run:

node --version

You should see a version number like v20.x.x.

4) Install Cursor (your AI-assisted editor)

Mental model: Cursor is a code editor with a built-in AI assistant. You’ll describe what you want in normal English, and it helps generate the code files.

  1. Download Cursor: cursor.com
  2. Install it and sign in if prompted.
  3. Open your project folder in Cursor (File → Open Folder).

5) Add docs to Cursor (so the AI has a cheat sheet)

Cursor is powerful, but it can sometimes suggest outdated patterns—especially for newer tools. Adding official docs gives it up-to-date reference material, which usually means fewer errors and faster progress.

  1. Open Cursor Settings (Ctrl+, on Windows).
  2. Go to Features → Docs.
  3. Click “Add Doc” and add these URLs:
https://www.instantdb.com/docs
https://nextjs.org/docs
https://tailwindcss.com/docs

After adding them, wait a minute for indexing to finish (Cursor will show it’s processing).
